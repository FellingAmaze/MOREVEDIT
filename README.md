### 2. Выгрузка активности

- [ ] Данные с подключённого источника отображаются на экране «Активность»
- [ ] Типы активности корректно классифицируются (шаги, бег, плавание, велосипед и т.д.)
- [ ] Данные обновляются при повторном входе на экран «Активность»
- [ ] Данные синхронизируются после перезапуска приложения
- [ ] Данные синхронизируются при подключении к интернету после офлайн-периода
- [ ] История активности отображается за выбранный период (день, неделя, месяц)
- [ ] Сбор данных происходит даже при неактивном/закрытом приложении (фоновая синхронизация)

### 3. Учёт активности

- [ ] Корректный подсчёт шагов
- [ ] Корректный подсчёт потраченных калорий
- [ ] Корректный подсчёт дистанции
- [ ] Корректный подсчёт минут активности
- [ ] Расчёт баллов здоровья на основе активности
- [ ] Данные в лидербордах (сообщество, соревнования, фонды, лиги) соответствуют фактической активности
- [ ] Учёт активности при использовании нескольких источников (приоритет: Встроенный шагомер → Рекомендованные устройства (Garmin, Polar, Suunto) → Google Fit, Health Connect, Apple Health)
- [ ] Приоритет источников соблюдается при конфликте данных
- [ ] Выбор типа активности перед тренировкой улучшает точность классификации
- [ ] Отображение нормального и максимального пульса (при наличии HRM)

### 4. Обработка ошибок

- [ ] Отсутствие интернета при подключении — отображение информативного сообщения
- [ ] Отказ в разрешениях (Google Fit / Apple Health) — обработка ошибки
- [ ] Неподдерживаемое устройство — сообщение пользователю
- [ ] Истекший токен авторизации Google Fit / Apple Health — повторный запрос авторизации
- [ ] Отсутствие данных на источнике — отображение нулевых значений без ошибок
- [ ] Потеря Bluetooth-соединения с HRM — отображение статуса «Устройство отключено»

### 5. Приватность и безопасность данных

- [ ] Настройки приватности по умолчанию: «Время моей активности» = ВКЛ (видно другим)
- [ ] Настройки приватности по умолчанию: «Мои фотографии» = ВКЛ (видно другим)

## Тест-кейсы для функционала авторизации/регистрации

### Тест-кейс: TC-AUTH-001 — Регистрация нового пользователя через Email

**Предусловие:** Приложение «Мой Фитнес» установлено, пользователь не авторизован

| Шаг | Действие | Ожидаемый результат |
|-----|----------|---------------------|
| 1 | Открыть приложение | Отображается экран входа/регистрации |
| 2 | Ввести валидный email (например, test@example.com) | Email принят, ошибок нет |
| 3 | Принять пользовательское соглашение и согласие на обработку ПД | Чекбоксы отмечены |
| 4 | Нажать «Зарегистрироваться» / «Продолжить» | Отправлено письмо с кодом подтверждения на указанный email |
| 5 | Перейти в почтовый ящик, открыть письмо | Письмо содержит код подтверждения или ссылку |
| 6 | Ввести полученный код подтверждения | Код принят, открывается экран ввода никнейма |
| 7 | Ввести уникальный никнейм | Никнейм принят |
| 8 | Пропустить подключение к фитнес-клубу | Отображается экран выбора уведомлений |
| 9 | Выбрать типы уведомлений и нажать «Продолжить» | Отображается экран ввода физических параметров |
| 10 | Ввести дату рождения, пол, вес, рост | Параметры приняты |
| 11 | Нажать «Сохранить» / «Готово» | Выполнен вход в приложение, отображается главный экран |
